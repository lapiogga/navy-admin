per D-14: MREG-01~04는 sys05 페이지를 Props(sysCode, 타이틀) 변경하여 직접 재사용.

sys05 페이지 재사용 방법: sys05 페이지가 sysCode prop을 받는지 확인. 받지 않으면 래퍼 컴포넌트로 감싸서 API 경로를 /sys06/으로 변경. 또는 sys05 페이지를 lazy import하여 직접 사용 (sys05 페이지가 URL 기반으로 sysCode를 추출하는 경우).

실제 구현: sys05 페이지 내부 코드를 read_first에서 확인 후, 재사용 가능한 방식을 선택.

Option A (sys05 페이지가 props로 sysCode를 받는 경우):
```tsx
import RegulationListPage from '@/pages/sys05-admin-rules/RegulationListPage'
// <Route path="1/1" element={<RegulationListPage sysCode="sys06" title="현행규정" />} />
```

Option B (sys05 페이지를 lazy import하는 경우):
```tsx
const RegulationListPage = lazy(() => import('@/pages/sys05-admin-rules/RegulationListPage'))
// <Route path="1/1" element={<Suspense fallback={<PageSpinner />}><RegulationListPage /></Suspense>} />
```

Option C (래퍼 컴포넌트):
```tsx
// 각 페이지 컴포넌트가 내부에서 /sys05/ 경로를 하드코딩한 경우,
// 래퍼에서 sysCode를 override하는 Context 또는 Props를 전달
```
