# 초과근무관리체계 (SYS01) 사용자 인터페이스 설계서

## 1. 설계 원칙

| 원칙 | 설명 |
|------|------|
| 일관성 | Ant Design 5 컴포넌트 기반, ProLayout 사이드바 레이아웃 통일 |
| 역할 분리 | 사용자 역할별 메뉴/버튼 노출 차등 (일반, 결재자, 관리자) |
| 상태 시각화 | StatusBadge, Steps, Tag로 처리 상태 직관적 표시 |
| 데이터 밀도 | ProTable/DataTable로 행정 목록 화면 최적화 |
| 접근성 | 군 행정 환경 특성상 데스크톱 브라우저 기준 설계 |

## 2. 공통 레이아웃

```
+--------------------------------------------+
|  Header (시스템명 + 사용자정보 + 로그아웃)   |
+------+-------------------------------------+
|      |                                     |
| Side |           Content Area              |
| bar  |   (Breadcrumb + PageHeader +         |
|      |    SearchForm + DataTable/Form)      |
|      |                                     |
+------+-------------------------------------+
```

- **Header**: 시스템명("초과근무관리체계"), 사용자 군번/계급/성명, 로그아웃 버튼
- **Sidebar**: ProLayout 사이드바, 7개 대메뉴 + 하위 메뉴 트리
- **Content Area**: Breadcrumb (경로 표시) + 각 화면 내용

## 3. 공통 UI 패턴

### 3.1 목록 화면 패턴 (CRUD List)

| 2  | ...  | ...  | ...  | Badge | [수정][삭제] |
+-------------------------------------------+
| Pagination: < 1 2 3 4 5 >                 |
+-------------------------------------------+

```
+-------------------------------------------+
| [Modal Title]                         [X] |
+-------------------------------------------+
| Label 1:  [Input / Select / DatePicker]   |

```
+-------------------------------------------+
