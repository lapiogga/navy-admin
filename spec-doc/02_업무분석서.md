## SYS01 초과근무관리체계 (99개 프로세스)

### 업무 개요
- 해병대 장병의 초과근무 신청/결재/정산/통계를 관리하는 체계. 신청서 작성부터 월말결산, 당직업무, 개인설정까지 초과근무 전 과정을 체계적으로 처리한다.
- 주요 사용자: 일반 사용자(신청자), 결재자, 부대관리자, 체계관리자, 당직자

### 핵심 업무 흐름
1. 신청서 관리 > 신청서 작성: 사용자가 초과근무 신청서를 작성하고 결재 요청
2. 신청서 관리 > 신청서 결재: 결재자가 신청서를 승인/반려
3. 신청서 관리 > 일괄처리: 일괄지급을 위한 일괄처리 신청서 작성/승인
4. 신청서 관리 > 월말결산: 부대관리자가 월별 초과근무 내역을 결산/마감
5. 현황조회 > 나의 근무현황/부재관리: 사용자의 연간/월간 근무현황 및 부재관리
6. 부대관리 > 최대인정시간/근무시간/공휴일/결재선: 부대 관리자의 설정 관리
7. 당직업무 > 당직개소/초과근무자/개인별 승인: 당직자 및 관리자의 당직업무
8. 개인설정 > 당직개소/부서 설정: 사용자의 개인 설정 관리

### 주요 기능

|--------|--------|----------|
| 게시판 | 공지사항, 질의응답 | 공통게시판 활용 |
| 신청서 관리 | 신청서 작성 | 목록/상세 조회, 신청서 작성/수정/삭제 |
| 신청서 관리 | 신청서 결재 | 결재대기 목록/상세 조회, 승인/반려 |
| 신청서 관리 | 일괄처리 | 일괄처리 신청서 CRUD |

- 입력: 신청서 종류(사전/사후), 근무일, 초과근무 시간(시작/종료), 신청자 IP, 당직개소, 초과근무 내용, 일괄처리 대상자/기간/지급시간/사유/근거자료
- 출력: 월말결산 엑셀, 초과근무 현황 그래프, 부대인원 엑셀, 승차권 등

### 특이사항/규칙
- 결재 이후 신청서 수정/삭제는 관리자만 가능

## SYS02 설문종합관리체계 (31개 프로세스)

### 업무 개요
- 해병대 내 설문조사를 체계적으로 관리하는 시스템. 설문 생성/문항 등록/승인/배포/참여/결과 분석까지 설문 전 주기를 관리한다.
- 주요 사용자: 설문 작성자(일반), 설문 참여자(일반), 체계관리자(승인/관리)

### 핵심 업무 흐름
1. 설문관리 > 나의 설문관리: 설문조사 설정 등록 > 문항 등록 > 제출(승인요청)
2. 설문관리 > 체계관리: 승인 대기 조회 > 설문조사 승인/반려
3. 설문관리 > 설문참여: 활성화된 설문 목록 조회 > 설문조사 실시
4. 설문관리 > 지난 설문보기: 기간 경과 설문 결과 조회/출력

### 주요 기능

| 대메뉴 | 소메뉴 | 주요 기능 |
|--------|--------|----------|
| 게시판 | 공지사항, 질의응답 | 공통게시판 활용 |
| 설문관리 | 나의 설문관리 | 목록/상세 조회, 설정 등록/수정, 문항 등록, 제출/삭제/재작성 |
| 설문관리 | 설문참여 | 목록/내용 조회, 설문조사 실시 |
| 설문관리 | 지난 설문보기 | 목록/결과 조회, 결과 엑셀 출력 |
| 설문관리 | 체계관리 | 승인 대기 목록/상세, 승인/반려, 승인 완료 목록/상세, 전체 설문관리 CRUD, 결과 조회/출력 |
| 관리자 | 시스템관리/결재/코드/게시판/권한 | 공통기능 모듈 적용 |

### 입력/출력 데이터
- 입력: 설문명, 설문내용(개요), 설문대상, 대상자 수, 설문기간, 결과공개 여부, 재응답 여부, 대상 계급/부대/직책/성별, 관련근거(승인공문), 설문지, 보안성검토 결과
- 출력: 설문 결과 차트(Chart 도구), 결과 엑셀

### 특이사항/규칙
- 설문조사 대상자 여부 체크 로직 필요
- 결과 공개 설정인 설문만 지난 설문 결과 조회 가능
- 문항 등록은 엑셀 양식 다운로드 후 업로드 방식, 데이터 오류 점검 필수

### 구현 페이지 (7개)
MySurveyPage, SurveyParticipationPage, PastSurveyPage, SurveyAdminPage, SurveyFormPage, SurveyQuestionEditor, SurveyResultPage, index

## SYS03 성과관리체계 (76개 프로세스)

### 업무 개요
- 해병대 부대(서)별 성과관리를 지원하는 체계. 기준정보 관리(기준년도/평가조직/과제), 연간과제 등록/실적 입력/승인/평가, 평가결과 조회까지 성과관리 전 주기를 관리한다.
- 주요 사용자: 과제 담당자(일반), 결재자, 평가자, 체계관리자

### 핵심 업무 흐름
1. 기준정보관리 > 시스템관리: 기준년도 CRUD

3. 기준정보관리 > 과제관리: 지휘방침 > 추진중점과제 > 중과제 계층 구조 관리

## SYS99 공통 기능 (82개 프로세스)

### 업무 개요
- 18개 서브시스템에서 공통으로 사용하는 관리 기능 모듈. 시스템관리(체계담당자/메뉴/메시지/로그), 결재관리, 코드관리, 공통게시판, 권한관리를 제공한다.
- 주요 사용자: 체계담당자(SuperAdmin), 각 서브시스템 관리자

### 핵심 업무 흐름
1. 시스템관리 > 체계담당자: SuperAdmin 관리 (조회/검색/추가/수정/삭제/저장)
2. 시스템관리 > 메뉴관리: 메뉴 트리 구조 관리 (조회/추가/수정)
3. 시스템관리 > 메시지 관리: 시스템 메시지 CRUD
4. 시스템관리 > 로그 조회: 장애로그/접속로그/엑셀다운로드로그 조회
5. 결재관리 > 결재선 관리: 결재선 CRUD/저장
6. 코드관리 > 코드그룹/코드: 코드체계 CRUD/저장
7. 공통게시판 > 게시판 설정: 게시판 CRUD, 게시글/카테고리/관리자/사용자/부대/첨부파일/설정/휴지통/URL 관리
